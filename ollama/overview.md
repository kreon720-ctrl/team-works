# TEAM WORKS 한 눈에 보기

## TEAM WORKS는 무엇인가요

TEAM WORKS는 팀 단위로 **일정·채팅·프로젝트·자료실을 한 화면에서** 관리하는 협업 도구입니다. 메신저로는 일정의 맥락을 잃고, 캘린더 도구로는 대화가 안 되며, 자료는 또 다른 곳에 흩어지는 문제를 한꺼번에 풉니다. 한 팀의 캘린더, 채팅, 공지, 업무보고, 포스트잇, 프로젝트 간트차트, 자료실(글+첨부), AI 버틀러까지 모두 같은 페이지에 있습니다.

## 누구를 위한 도구인가요

- **소규모 팀(3~30명)**의 협업 — 학교 동아리, 스타트업, 사내 TF, 프리랜서 협업 그룹
- 별도 채팅 앱 + 캘린더 + 노션 + 드라이브를 각각 쓰는 게 부담스러운 팀
- "오늘 누가 뭐 하는지" + "다음 주 일정" + "그 회의 자료" 가 한 화면에 보였으면 하는 팀

## 핵심 컨셉 6가지

1. **팀 단위 격리** — 한 팀의 일정·채팅·포스트잇·프로젝트·자료실은 그 팀 안에서만 보입니다. 다른 팀에는 절대 노출되지 않습니다.
2. **채팅방 컨텍스트 격리** — 채팅·공지·자료실은 **(팀, 프로젝트)** 조합으로 분리됩니다. 일자별 팀 채팅과 프로젝트 전용 채팅이 서로 안 섞입니다.
3. **생성자 기반 권한** — 내가 만든 것만 내가 수정/삭제합니다. 팀장이라도 다른 사람의 일정·자료실 글을 임의로 고치지 못합니다.
4. **공개 팀 + 가입 신청** — 다른 팀에 들어가려면 공개 팀 목록 → 가입 신청 → 팀장 승인 절차를 거칩니다.
5. **업무보고 ≠ 일반 채팅** — 평가성 보고는 별도 메시지 타입(WORK_PERFORMANCE)으로, 기본은 팀장만 봅니다.
6. **AI 버틀러 "찰떡"** — 단일 입력창에서 자연어를 받아 자동으로 4가지(사용법·일반·일정 조회·일정 등록) 중 적절한 길로 보냅니다. 모드 직접 선택 불필요.

## 주요 기능 목록

| 기능 | 설명 |
|------|------|
| 회원가입 / 로그인 | 이메일·비밀번호 + JWT (Access 15분 / Refresh 7일) |
| 팀 만들기·수정·삭제 | 팀명·설명·공개 여부. 만든 사람이 자동으로 팀장 |
| 팀 탐색 / 가입 신청 | 공개 팀 목록 → 가입 신청 → 팀장 승인 |
| 나의 할 일 | 내가 팀장인 모든 팀의 PENDING 가입 신청을 한 번에 처리 |
| 팀 캘린더 | 월·주·일 뷰. 일정 추가·수정·삭제 |
| 포스트잇 | 날짜에 붙이는 메모. 5가지 색상 |
| 팀 채팅 (일자별) | NORMAL · WORK_PERFORMANCE 두 타입. KST 날짜별 그룹핑 |
| 프로젝트 전용 채팅방 | 프로젝트별 독립 채팅 + 공지 + 자료실 (일자별과 격리) |
| 업무보고 | WORK_PERFORMANCE 타입. 팀장 / 권한 받은 팀원만 조회 |
| 공지사항 | 채팅 상단 고정 알림. 팀 일자별 / 프로젝트별 격리 |
| 프로젝트 / 간트차트 | 프로젝트 → 프로젝트 일정 → 세부 일정 3단계 |
| **자료실(게시판)** | 채팅방 sub-탭. 글 + 첨부파일(≤10MB). (팀, 프로젝트) 격리 |
| AI 버틀러 (찰떡) | 4-way 자동 분류 (사용법 / 일반 / 일정 조회 / 일정 등록). SSE 스트리밍 + 다중 턴 |

## 사용자 역할 두 가지

**팀장 (LEADER)**
- 팀을 만든 사람. 팀당 1명.
- 가입 신청 승인/거절 가능
- 업무보고 열람 권한을 다른 팀원에게 부여 가능
- 팀 정보 수정·팀 삭제·팀원 강제 탈퇴 가능 (본인 제외)
- 자신이 만든 콘텐츠만 수정/삭제 가능 — 단, **공지사항은 팀장 권한으로 다른 사람 것도 삭제 가능**

**팀원 (MEMBER)**
- 팀에 가입된 일반 사용자
- 팀 일정·채팅·포스트잇·프로젝트·공지·자료실 글 모두 작성 가능
- 자신이 만든 콘텐츠만 수정/삭제 가능
- 업무보고는 팀장이 권한 부여한 경우에만 다른 팀원의 보고를 열람 가능

## 한 사용자가 여러 팀에 속할 수 있나요

네. 한 계정으로 여러 팀의 LEADER가 될 수도 있고, 어떤 팀에서는 LEADER, 다른 팀에서는 MEMBER일 수도 있습니다. **나의 할 일** 메뉴에서 내가 팀장인 모든 팀의 PENDING 가입 신청을 한 번에 처리할 수 있습니다.

## 운영 환경

- 모든 서비스가 **Docker Compose** 단일 호스트에서 컨테이너로 동작합니다 — Postgres / Backend / Frontend / Nginx / Open WebUI / SearxNG.
- 자료실 첨부파일은 1단계에서 호스트 디렉토리에 저장되며, 운영 전환 시 환경 변수 한 줄(`STORAGE_BACKEND=s3`) 토글로 클라우드 스토리지로 무중단 이전 가능합니다.
- AI 버틀러는 로컬 Ollama(gemma4:26b) + RAG 인덱스 + Open WebUI + SearxNG 메타검색을 조합합니다. 사용법 답변은 외부 호출 없이 로컬에서 처리됩니다.
