# AI 버틀러 "찰떡"

## 찰떡은 어디에서 호출하나요

팀 메인 화면(`/teams/[팀ID]`) 우측 채팅 영역 상단에 **팀채팅 / AI 버틀러** 두 탭이 있습니다. **AI 버틀러** 탭을 누르면 단일 입력창 + 대화 영역이 표시됩니다. 이전에 별도 작은 팝업창으로 떴던 형태에서 우측 영역에 통합되었습니다.

## 어떻게 쓰나요 — 단일 입력창

찰떡은 **모드를 직접 선택할 필요가 없습니다**. 입력창에 자연어로 질문이나 요청을 쓰면 시스템이 자동으로 의도를 분류해 적절한 길로 보냅니다. 사용자는 "내가 지금 무얼 묻는지" 신경 쓸 필요 없이 자유롭게 입력하세요.

## 4-way 자동 의도 분류

찰떡은 입력을 받자마자 다음 5개 중 하나로 자동 분류합니다:

| 분류 | 언제 | 처리 경로 | 답변에 표시되는 것 |
|------|------|---------|------|
| **사용법 (usage)** | "포스트잇 색깔 종류", "프로젝트 등록하는 법", "업무보고가 뭐예요?" | RAG — TEAM WORKS 공식 문서(`ollama/*.md`) 검색 후 답변 | 📚 **공식 문서 N건 참조** 출처 뱃지 |
| **일반 (general)** | "오늘 서울 날씨", "오늘 뉴스 검색", "Next.js 16 새 기능", 일반 잡담 | SearxNG 메타검색(Google·Bing·Naver·DuckDuckGo) 5건 → Open WebUI(gemma4:26b) | 🌐 **웹 검색 N건 참조** 출처 URL |
| **일정 조회 (schedule_query)** | "오늘 일정 알려줘", "이번 주 회의", "디자인 리뷰 언제야?" | backend Schedule API 직접 조회 → 코드가 한국어로 즉시 포맷 (LLM 답변 본문 0회) | 즉시 응답. 매칭 0건이면 자동으로 더 넓은 범위 재조회 |
| **일정 등록 (schedule_create)** | "내일 오후 3시 주간회의 등록해줘" | 자연어를 인자(title/startAt/endAt)로 파싱 → **confirm 카드** 송출 | ✓ 클릭해야 INSERT. 정보 부족 시 후속 질문(다중 턴) |
| **거절 안내 (blocked)** | "어제 회의 삭제해줘", "프로젝트 등록해줘" 등 일정 조회·등록 외 변경 요청 | 정중한 거절 안내 | "찰떡이는 일정 조회·등록만 도와드릴 수 있어요" 안내 |

분류 결과는 사용자에게 별도로 보이지 않습니다 — 결과(답변·confirm 카드 등)만 보입니다.

## 사용법(usage)이란 어떤 질문인가요

TEAM WORKS 자체 사용법에 대한 질문입니다. 찰떡이 RAG 인덱스(공식 안내 문서들)를 검색해 출처와 함께 답합니다.

예시:
- "팀에 어떻게 가입해?"
- "업무보고 메시지가 뭐예요?"
- "포스트잇 색깔 종류 알려줘"
- "프로젝트 삭제하면 하위 일정도 같이 삭제되나요?"
- "다른 사람이 만든 일정 수정할 수 있어?"
- "팀장과 팀원의 차이가 뭔가요?"
- "자료실에 첨부파일 몇 MB까지 올라가요?"

답변 카드 하단에 **📚 공식 문서 N건 참조** 라는 출처 뱃지가 붙고, 어떤 문서의 어느 섹션을 참고했는지 인용 정보가 표시됩니다.

## 일반(general)이란 어떤 질문인가요

TEAM WORKS와 무관한 시사·날씨·코딩·일반 지식 등입니다. 찰떡이 SearxNG 메타검색 엔진(Google/Bing/Naver/DuckDuckGo 등)에서 5건을 가져와 모델 컨텍스트에 넣고 Open WebUI의 gemma4:26b로 답합니다.

예시:
- "오늘 서울 날씨 어때?"
- "오늘 뉴스 검색해줘"
- "Next.js 16 에서 새로 추가된 기능은?"
- "파이썬에서 리스트 정렬하는 방법"

답변 카드 하단에 **🌐 웹 검색 N건 참조** 와 함께 참조한 URL 5건이 표시됩니다.

## 일정 조회(schedule_query)는 어떻게 동작하나요

자연어 일정 조회 요청을 받으면 찰떡이 `{view, date, keyword}` 로 파싱한 뒤 backend Schedule API를 직접 호출합니다. 결과는 **코드가 한국어로 포맷** 해서 즉시 송출 — LLM이 답변 본문을 다시 생성하지 않으므로 빠릅니다.

예시 흐름:
- 입력: "오늘 일정 알려줘"
- 처리: `view=day, date=오늘 KST` 로 파싱 → 팀 일정 GET → "기획팀 팀의 2026-04-30 일정 N건: ..." 즉시 출력
- 매칭 0건일 때: 키워드가 있으면 자동으로 month 범위로 fallback 재조회

다른 팀의 일정은 보이지 않습니다 — 백엔드 미들웨어가 사용자의 팀 멤버십을 검증하므로 보안적으로 격리됩니다.

## 일정 등록(schedule_create)은 어떻게 동작하나요

자연어 일정 등록 요청을 받으면 찰떡이 인자(`title`, `startAt`, `endAt`)를 추출한 뒤 **confirm 카드**를 띄웁니다. 사용자가 ✓ 클릭해야 실제로 등록되며, 클릭 전에는 DB가 변경되지 않습니다.

기본 흐름:
1. 입력: "내일 오후 3시 주간회의 등록해줘"
2. 찰떡이 `{title:'주간회의', startAt:'내일 15:00', endAt:'16:00'}` 로 파싱
3. **confirm 카드** 표시 — "이 내용으로 등록할까요?" + 미리보기
4. 사용자가 ✓ → 등록 완료 + 좌측 캘린더 자동 갱신
5. 사용자가 ✗ → "실행을 취소했어요" 안내, DB 변경 없음

## 정보가 부족한 등록 요청은 어떻게 되나요 (다중 턴)

시간이나 제목이 빠진 요청을 받으면 찰떡이 후속 질문을 던집니다 (`awaiting-input` 상태).

예시:
1. 입력: "내일 회의 등록해줘" (시간 미명시)
2. 찰떡: "몇 시에 잡을까요?" + `awaiting-input` 표시
3. 사용자: "오후 3시"
4. 찰떡이 직전 미완 질문과 결합해 재요청 → confirm 카드 표시
5. 사용자가 ✓ → 등록 완료

정보가 모일 때까지 여러 번 주고받을 수 있습니다.

## 거절 안내(blocked)는 무엇인가요

찰떡이 처리할 수 없는 요청을 받으면 정중히 거절합니다.

거절 대상:
- **일정 수정·삭제** — "어제 회의 삭제해줘", "내일 회의를 모레로 옮겨줘"
- **다른 도메인 변경** — "프로젝트 등록해줘", "공지 올려줘", "포스트잇 추가해줘", "자료실 글 써줘"

답변:
> "찰떡이는 **일정 조회·등록** 만 도와드릴 수 있어요. 일정 수정·삭제는 캘린더에서 직접 처리해 주세요. 🙏"

또는

> "프로젝트·채팅·공지·포스트잇 같은 작업은 화면에서 직접 처리해 주세요."

## 사용법 시그널이 함께 있는 경우

"프로젝트 등록하는 법 알려줘" 같이 변경 요청처럼 보이지만 실제로는 사용법 질문인 경우, 찰떡이 사용법 키워드("…법", "어떻게")를 우선 매칭해서 거절이 아닌 RAG 사용법 답변으로 보냅니다.

## 답변은 얼마나 빨리 도착하나요 (SSE 스트리밍)

답변은 SSE(Server-Sent Events)로 토큰 단위로 점진 표시됩니다. 첫 토큰이 약 3~10초 안에 도착해 빈 카드 시간을 최소화합니다. 일정 조회는 코드가 직접 포맷하므로 거의 즉시 응답합니다.

## 변경 작업은 즉시 실행되나요

아니요. **일정 등록**은 confirm 카드 → 사용자 ✓ 클릭 후에만 실행됩니다. **일정 조회**는 즉시 실행됩니다(읽기 전용이라 안전). **사용법·일반 질문**은 외부 시스템에 변경을 가하지 않습니다.

## 답변이 부정확하면 어떻게 하나요

찰떡은 LLM 기반이라 가끔 부정확하거나 오래된 정보를 답할 수 있습니다. 다음을 확인해 보세요:
- 답변에 첨부된 **참고 문서 출처(📚)** 또는 **웹 검색 출처(🌐)** 를 직접 열어 사실 확인
- 사용법 안내가 실제 화면과 다르다면 화면을 우선 신뢰
- 일정 조회 결과가 이상하면 직접 캘린더에서 다시 확인

심각한 오답은 운영자에게 알려주세요. RAG 인덱스 갱신 또는 분류기 보정으로 개선합니다.

## 어떤 모델을 사용하나요

- **답변 생성**: Ollama의 **gemma4:26b** (호출 시점에 `num_ctx 32K`)
- **임베딩(RAG 검색)**: **nomic-embed-text** (별도 모델, 역할 분리)
- **일반 질문 경로**: 같은 gemma4:26b 모델을 Open WebUI의 **gemma4-web** 프리셋(웹 검색 활성화)으로 호출

모델·임베딩 교체 시 RAG 인덱스를 재빌드해야 합니다.

## 찰떡은 내 데이터를 외부로 보내나요

- **사용법 질문 (📚 RAG 경로)**: 처리 전부가 로컬 Ollama에서 수행. 외부 전송 없음.
- **일반 질문 (🌐 웹 검색 경로)**: 검색 쿼리만 SearxNG 메타검색 엔진을 거쳐 외부 검색 엔진으로 나갑니다. 답변 생성은 다시 로컬에서 수행됩니다 — **검색 쿼리만 외부, 답변 생성은 로컬**.
- **일정 조회·등록**: 백엔드 API 호출만. 외부 LLM 서비스로 데이터가 가지 않습니다.

어느 경로든 답변·검색 쿼리·결과는 OpenAI/Anthropic 같은 외부 LLM API로 전송되지 않습니다.

## DB 안전성은 어떻게 보장되나요

찰떡은 **SQL을 자유롭게 생성하지 않습니다**. AI 모델은 자연어를 JSON 인자로 변환만 하고, 실제 DB 접근은 backend의 검증된 SQL 템플릿을 통과합니다. 모든 호출은 backend 미들웨어(`withAuth` + `withTeamRole`)로 권한이 격리되어, 다른 팀의 데이터에 접근할 수 없습니다.

## 한 번에 얼마나 긴 질문을 받나요

컨텍스트 윈도우가 32K입니다. 일반 사용에서는 길이 제한을 신경 쓰지 않아도 됩니다. 다만 답변 품질은 질문이 구체적일수록 좋습니다 — "팀 어떻게 만들어?"보다 "공개 팀 만들어서 다른 사람이 가입 신청하게 하려면 어떻게 해?" 가 더 정확한 답을 받습니다.

## 찰떡 이름은 왜 "찰떡"인가요 / 이름 유래

> 개떡같이 물어도 찰떡같이 대답하라고 주인님이 지어주셨어요~~

질문이 다소 두루뭉술하거나 정리가 덜 되어 있어도, 의도를 알아채서 딱 맞는 답을 돌려드리겠다는 다짐이 담긴 이름입니다. AI 비서·AI 버틀러·찰떡이는 모두 같은 도구를 가리킵니다.
