# AI 비서 "찰떡"

## AI 비서는 어디에서 호출하나요

두 가지 방법이 있습니다:
1. 직접 `/ai-assistant` URL로 접속
2. 팀 메인 화면 상단의 **AI 버틀러** 아이콘 버튼 클릭 → 별도 작은 창(480×720)으로 팝업

팝업으로 뜨는 이유는 작업 화면을 가리지 않기 위함입니다. 만약 전체 화면으로 떠버린다면 브라우저의 팝업 차단 설정 또는 확장 프로그램을 확인해보세요.

## 두 가지 모드는 무엇인가요

찰떡은 두 모드 중 하나로 동작합니다:

**안내 모드 (Guide)** — 사용법 + 일반 질문 모두 답변
- TEAM WORKS **사용법 질문**: RAG(Retrieval-Augmented Generation) 로 공식 안내 문서를 검색해 답변. 답변 카드 하단에 📚 **참고 문서 출처** 가 표시됩니다.
- **일반 질문**(시사·날씨·코딩·일반 지식 등): TEAM WORKS 문서에 없는 질문이면 자동으로 **웹 검색(SearxNG)** 결과를 조합해 답변. 답변 카드 하단에 🌐 **참조 웹 출처 URL** 이 표시됩니다.
- 비로그인 상태에서도 사용 가능
- 라우팅은 자동: 시스템이 질문 키워드와 RAG 답변의 거절 시그널을 보고 사용법/일반 중 적합한 경로로 보냅니다
- 실제 데이터를 조회·생성하지 않습니다 (순수 정보 제공)

**실행 모드 (Agent)**
- 실제 데이터 **조회 및 등록**을 자동화합니다
- 로그인 필수 (인증 토큰 필요)
- MCP(Model Context Protocol) 도구를 호출해 백엔드 API와 통신
- 변경 작업(일정 등록 등) 전에 사용자에게 확인 카드를 보여줍니다
- 사용자가 승인하면 그때 실행합니다

## 안내 모드에서 무엇을 물어볼 수 있나요

크게 두 종류 모두 가능합니다:

**A. TEAM WORKS 사용법** — 공식 문서를 검색해 정확한 절차·버튼 이름을 답변. 답변 하단에 📚 참고 문서가 함께 표시됩니다.
- "팀에 어떻게 가입해?"
- "업무보고 메시지가 뭐예요?"
- "포스트잇 색깔 종류 알려줘"
- "프로젝트 삭제하면 하위 일정도 같이 삭제되나요?"
- "다른 사람이 만든 일정 수정할 수 있어?"
- "팀장과 팀원의 차이가 뭔가요?"

**B. 일반 질문** — 시사·날씨·코딩·지식 등 TEAM WORKS 와 무관한 질문이어도 OK. 시스템이 자동으로 웹 검색을 거쳐 답변합니다. 답변 하단에 🌐 참조 웹 출처 URL 이 표시됩니다.
- "오늘 서울 날씨 어때?"
- "오늘 뉴스 검색해줘"
- "Next.js 16 에서 새로 추가된 기능은?"
- "파이썬에서 리스트 정렬하는 방법"

라우팅은 자동입니다 — 사용자가 모드를 따로 고를 필요 없이, 안내 모드에서 자유롭게 두 종류를 섞어 물어보면 됩니다. 시스템이 질문 성격을 판단해 사용법은 RAG, 일반은 웹 검색으로 보냅니다.

## 실행 모드에서 무엇을 시킬 수 있나요

현재 구현된 작업 3가지:

**1) 내 팀 목록 보여줘**
- 호출되는 도구: `listMyTeams`
- 결과: 사용자가 속한 모든 팀의 이름·역할 목록

**2) 특정 팀의 일정 보여줘 (월간 단위)**
- 호출되는 도구: `listTeamSchedules`
- 입력: 팀ID, 날짜(YYYY-MM-DD)
- 결과: 그 월의 모든 일정

**3) 새 일정 등록해줘**
- 호출되는 도구: `createSchedule`
- 입력: 팀ID, 제목, 시작 시각, 종료 시각, (선택) 설명
- 동작: 등록 전 확인 카드를 보여주고, 사용자 [승인] 후 실제 생성

자연어로 부탁하면 찰떡이 적절한 도구와 파라미터를 골라 실행합니다. 예: "내일 오후 3시 주간회의 일정 등록해줘" → `createSchedule` 호출 준비 → 확인 카드 표시.

## 실행 모드가 못 하는 것은 무엇인가요

현재 MVP 버전에서는 다음 작업을 **할 수 없습니다**:
- 일정 수정·삭제
- 채팅 메시지 송수신
- 포스트잇 작성/수정/삭제
- 공지사항 작성
- 프로젝트·간트 관리
- 팀 만들기·가입 신청
- 가입 신청 승인/거절

이런 작업이 필요하면 사용자가 직접 화면에서 수행해야 합니다. 향후 도구 확장이 예정되어 있습니다.

## 등록 작업은 즉시 실행되나요

아니요. 등록·생성·수정 작업은 **확인 카드**가 먼저 뜹니다. 카드에는 "이 내용으로 등록할까요?"라는 질문과 함께 미리보기가 표시되며, 사용자가 [승인]을 눌러야 실제로 실행됩니다. 조회 작업(목록 보기, 일정 보기)은 즉시 실행됩니다.

## 답변이 부정확하면 어떻게 하나요

찰떡은 LLM 기반이라 가끔 부정확하거나 오래된 정보를 답할 수 있습니다. 다음을 확인해 보세요:
- 답변에 첨부된 **참고 문서 출처**를 직접 열어 사실 확인
- 사용법 안내가 실제 화면과 다르다면 화면을 우선 신뢰
- 데이터 조회 결과가 이상하면 직접 화면에서 다시 조회

심각한 오답은 운영자에게 알려주세요. RAG 인덱스 갱신 또는 도구 보정으로 개선합니다.

## 어떤 모델을 사용하나요

현재 로컬 환경에서 **Ollama**의 **gemma4:26b** 모델을 사용합니다(호출 시점에 `num_ctx 32768` 컨텍스트 윈도우로 동작). 검색용 임베딩은 별도 모델 **nomic-embed-text** 가 담당합니다(역할이 다릅니다). 모델·임베딩 모델 교체 시 RAG 인덱스를 다시 빌드해야 합니다.

일반 질문 경로에는 같은 `gemma4:26b` 모델이지만 Open WebUI 의 **`gemma4-web` 프리셋**(웹 검색 활성)으로 호출되어, 검색 결과를 시스템이 추가 컨텍스트로 모델에 전달합니다.

## 찰떡은 내 데이터를 외부로 보내나요

- **사용법 질문(📚 RAG 경로)**: 처리 전부가 로컬 Ollama 에서 수행. 외부로 데이터 전송 없음.
- **일반 질문(🌐 웹 검색 경로)**: 질문 키워드가 SearxNG 메타 검색 엔진을 거쳐 Google·Bing·DuckDuckGo·Brave·Naver 등 외부 검색 엔진으로 나갑니다. 검색 결과 페이지의 본문 일부도 가져와 모델 컨텍스트로 사용합니다. 답변 자체는 다시 로컬 Ollama 에서 생성합니다 — 즉 **검색 쿼리만 외부로 나가고 답변 생성은 로컬**.
- 어느 경로든 답변·검색 쿼리·결과는 OpenAI/Anthropic 같은 외부 LLM API 로는 전송되지 않습니다.

## 한 번에 얼마나 긴 질문을 받나요

컨텍스트 윈도우가 128K로 매우 넓습니다. 일반 사용에서는 길이 제한을 신경 쓰지 않아도 됩니다. 다만 답변 품질은 질문이 구체적일수록 좋습니다 — "팀 어떻게 만들어?"보다 "공개 팀 만들어서 다른 사람이 가입 신청하게 하려면 어떻게 해?" 가 더 정확한 답을 받습니다.

## 찰떡 이름은 왜 "찰떡"인가요 / 이름 유래 / 왜 찰떡이라고 부르나요

> 개떡같이 물어도 찰떡같이 대답하라고 주인님이 지어주셨어요~~

질문이 다소 두루뭉술하거나 정리가 덜 되어 있어도, 의도를 알아채서 딱 맞는 답을 돌려드리겠다는 다짐이 담긴 이름입니다. AI 비서·AI 버틀러·찰떡이는 모두 같은 도구를 가리킵니다.
