# 채팅 · 업무보고 · 공지사항

## 채팅은 어디에서 보나요

팀 메인 화면(`/teams/[팀ID]`)의 오른쪽에 채팅 영역이 있습니다. 왼쪽 캘린더와 동시에 표시됩니다.

우측 채팅 영역에는 두 단계의 탭이 있습니다:
- 상단 **팀채팅 / AI 버틀러** — 팀원들과의 대화 vs 찰떡과의 대화
- 팀채팅 안 sub-탭 **채팅 / 자료실** — 메시지 vs 자료실 글

캘린더의 날짜를 클릭하면 채팅이 그 날짜의 메시지로 자동 전환됩니다. 일정과 대화의 맥락을 한 화면에서 잇기 위한 디자인입니다.

## 일자별 채팅과 프로젝트 채팅의 차이

채팅은 **(팀, 프로젝트) 격리** 패턴으로 두 가지 컨텍스트를 가집니다.

- **팀 일자별 채팅** (`projectId NULL`) — 좌측 캘린더가 월·주·일 뷰일 때 우측에 표시. KST 날짜 단위로 그룹핑.
- **프로젝트 전용 채팅** (`projectId 채워짐`) — 좌측이 프로젝트 간트차트 뷰일 때 우측이 그 프로젝트 채팅방으로 자동 전환. 같은 `projectId`의 모든 메시지가 시간 순서로 표시.

두 컨텍스트는 서로 섞이지 않습니다. 프로젝트 A의 메시지가 팀 일자별 채팅이나 프로젝트 B 채팅방에 보이지 않습니다. 공지·자료실도 같은 격리 패턴을 따릅니다.

## 일반 메시지(NORMAL)는 어떻게 보내나요

1. 채팅 입력창에 메시지를 입력합니다.
2. **전송** 또는 Enter 키로 보냅니다.
3. 즉시 채팅 영역에 추가되고, 같은 채팅방의 팀원 모두에게 보입니다.
4. 메시지는 **최대 2000자**까지 가능합니다.

## 업무보고(WORK_PERFORMANCE)는 무엇인가요

팀원이 팀장에게 올리는 평가성·민감한 보고를 위한 별도 메시지 타입입니다. 일반 채팅과 같은 입력창에서 보내지만 메시지 타입을 "업무보고"로 선택해서 보냅니다 (UI에 토글/선택지 있음).

업무보고의 특별한 점:
- **기본적으로 팀장만 봅니다** — 다른 팀원은 보이지 않음
- 팀장이 명시적으로 권한을 부여한 팀원만 다른 사람의 업무보고를 열람 가능
- 작성한 본인은 항상 자기 메시지를 볼 수 있음
- 다른 팀원이 훔쳐볼 수 없는 안전한 구조

이렇게 분리한 이유: 평가나 솔직한 의견이 다른 팀원에게 노출되면 솔직히 말하기 어렵기 때문입니다.

## 업무보고는 누가 만들 수 있나요

팀 구성원 누구나(LEADER·MEMBER 모두) 보낼 수 있습니다. 다만 일반적인 사용은 MEMBER가 LEADER에게 보고하는 형태입니다.

## 업무보고 열람 권한은 어떻게 설정하나요

**팀장(LEADER)만** 설정할 수 있습니다.

1. 팀 화면에서 **업무보고 조회 권한 설정** 메뉴(또는 채팅 영역 우상단 [관리자] 버튼)를 누릅니다.
2. 현재 팀의 모든 팀원 목록이 표시되고, 각자 권한 부여 여부 체크박스가 있습니다.
3. 권한을 줄 팀원을 체크하고 **저장** → 즉시 적용됩니다.
4. 권한이 부여된 팀원은 그 팀의 모든 업무보고 메시지를 열람할 수 있습니다.

빈 상태(아무도 권한 없음)면 팀장만 업무보고를 봅니다.

## 공지사항(Notice)은 무엇인가요

채팅 상단에 **고정**되어 표시되는 알림입니다. "내일 오전 10시 전체 회의" 같은 중요한 안내용입니다. 일반 메시지는 시간순으로 흘러가지만 공지는 항상 최상단에 보입니다.

공지의 특징:
- 채팅 영역 최상단에 고정
- 최대 2000자
- 여러 개의 공지를 동시에 가질 수 있음
- 팀원 누구나 작성 가능
- 삭제는 작성자 본인 또는 팀장만 가능 (팀장은 팀원이 쓴 공지도 삭제할 수 있음 — 예외 권한)
- 일자별 / 프로젝트별 격리 — 채팅과 동일

## 공지사항은 어떻게 작성하나요

1. 채팅 영역의 **공지 작성** 버튼을 누릅니다.
2. 공지 내용을 입력합니다 (최대 2000자).
3. **등록** → 채팅 상단에 즉시 고정됩니다.

좌측 캘린더 컨텍스트에 따라 자동으로 일자별 공지 또는 프로젝트 전용 공지로 등록됩니다.

## 채팅 메시지는 영구 보관되나요

네. 모든 메시지(일반·업무보고·공지)는 DB에 영구 저장됩니다. 시간이 지나도 사라지지 않습니다. 다만 메시지 삭제 기능은 현재 MVP 버전에 제한적입니다.

## 채팅 메시지는 실시간으로 도착하나요

WebSocket 기반의 완전 실시간은 아니지만, 화면이 **3초마다 자동 갱신** 되어 새 메시지가 빠르게 보입니다. 보내는 즉시 본인 화면에는 반영(optimistic update)되므로 입력 흐름이 끊기지 않습니다.

## 다른 팀의 채팅도 보이나요

아니요. 한 팀의 채팅·업무보고·공지·자료실은 그 팀 안에서만 노출됩니다. 팀 간 채널 공유는 없습니다.

## 채팅에 파일이나 이미지를 올릴 수 있나요

채팅 메시지 자체에는 첨부파일 기능이 없습니다. 다만 **자료실** sub-탭(같은 채팅방 컨텍스트)에서 글 + 첨부파일(≤10MB) 형태로 자료를 올릴 수 있습니다. 자료실은 일자별 / 프로젝트별로 격리되어 있어 채팅처럼 자동 분류됩니다.

## 채팅 메시지에 멘션(@username)이나 이모지가 가능한가요

현재 MVP 버전에는 별도 멘션·이모지 기능이 없습니다. 일반 텍스트만 지원합니다 (이모지 문자 자체는 입력 가능).

## 채팅 검색이 가능한가요

현재 MVP 버전에는 채팅 텍스트 검색 기능이 없습니다. 캘린더 날짜 선택으로 그 날의 메시지를 좁혀 보거나, 프로젝트별 채팅방으로 컨텍스트를 좁히는 방식입니다. 향후 추가 예정 기능입니다.
