# 프로젝트 · 간트차트 · 생성(등록·추가·만들기) · 수정(편집·변경) · 삭제(제거)

> 검색 키워드: 프로젝트 생성, 등록, 추가, 만들기, 만드는법, 만드는 방법, 신규프로젝트, 프로젝트 일정, 세부일정, 간트차트, 진행률, 지연, 단계, phase, 담당자, 매니저, 프로젝트 수정, 편집, 변경, 프로젝트 삭제, 제거, 지우기, CASCADE, 간트차트 저장, SVG, 다운로드, 내보내기

## 프로젝트는 무엇인가요

프로젝트는 업무의 **큰 틀** 입니다. "2분기 신제품 런칭", "사이트 리뉴얼", "여름 워크샵 준비" 같은 단위로 만듭니다. 프로젝트 안에는 **프로젝트 일정**(간트차트의 한 줄) 을 두고, 그 아래 다시 **세부 일정** 을 넣는 3단계 구조입니다.

```
프로젝트 (Project)
 └── 프로젝트 일정 (ProjectSchedule) ← 간트차트의 한 줄
      └── 세부 일정 (SubSchedule)    ← 들여쓰기로 표시
```

각 프로젝트는 **독립된 채팅방·공지·자료실** 을 가집니다. 프로젝트를 열면 우측 채팅 영역이 그 프로젝트 전용 컨텍스트로 자동 전환되어, 일자별 팀 채팅과는 따로 관리됩니다.

## 프로젝트 페이지로 들어가기

팀 메인 화면(`/teams/[팀ID]`) 캘린더 상단의 **`프로젝트 관리`** 버튼을 누릅니다. 좌측에 프로젝트 목록(드롭다운 또는 탭) 과 간트차트가 표시됩니다. 우측 채팅은 선택된 프로젝트 전용 채팅방으로 자동 전환됩니다.

프로젝트 목록 우측 액션 아이콘:
- **`프로젝트 생성`** (➕) — 새 프로젝트 생성
- **`프로젝트 수정`** (연필) — 현재 선택된 프로젝트 수정
- **`프로젝트 삭제`** (휴지통) — 현재 선택된 프로젝트 삭제
- **`+일정`** — 현재 프로젝트에 프로젝트 일정 추가

## 프로젝트 생성(등록·추가·만들기) — 단계별

프로젝트를 **생성**(등록·추가·만들기) 하려면 다음 절차를 따릅니다.

1. 프로젝트 페이지 좌측 상단의 **`프로젝트 생성`** ➕ 아이콘을 누릅니다.
2. **"프로젝트 생성"** 모달이 열립니다. 다음을 입력합니다:
   - **이름** (필수, 최대 200자) — 예: "2분기 신제품 런칭"
   - **설명** (선택)
   - **시작 날짜 / 종료 날짜** (종료 ≥ 시작, 같은 날짜 허용)
   - **담당자(매니저)** — 팀원 중 선택
   - **단계 (phases)** — "기획 → 개발 → 출시" 같은 자유 배열 (쉼표·줄바꿈으로 구분)
   - **진행률** (0~100%) — 초기값은 보통 0
3. **`생성`** 버튼 → 프로젝트 목록에 추가되고, 자동으로 그 프로젝트가 선택됩니다.

### 단계(phases) 의 의미
프로젝트가 어떤 흐름으로 진행되는지 사용자가 정의하는 자유 배열입니다. 예시:
- 신제품 런칭: "기획 → 디자인 → 개발 → QA → 출시"
- 워크샵: "장소 섭외 → 프로그램 기획 → 참가자 모집 → 운영"

단계는 시각적 구분용이며, 실제 일정 진행은 그 아래 **프로젝트 일정** 으로 관리합니다.

## 프로젝트 일정 추가(등록·생성·만들기) — 단계별

프로젝트 일정을 **추가**(등록·생성·만들기) 하려면 다음 절차를 따릅니다. 프로젝트 일정은 간트차트의 한 줄(row) 입니다.

1. 프로젝트를 선택한 상태에서 우측의 **`+일정`** 버튼을 누릅니다.
2. **"프로젝트 일정 생성"** 모달:
   - **제목** (필수, 최대 200자) — 예: "UI/UX 디자인"
   - **시작·종료 날짜** (종료 ≥ 시작)
   - **진행률** (0~100%)
   - **지연 여부 (isDelayed)** — 체크박스
   - **담당자 (leader)**
   - **색상** (5가지 — `indigo` / `blue` / `emerald` / `amber` / `rose`)
   - **단계 (phase)** — 위에서 정의한 단계 중 선택
3. **`생성`** → 간트차트에 새 줄로 추가, 진행률만큼 막대가 채워집니다.

## 세부 일정 추가(등록·생성·만들기) — 단계별

세부 일정을 **추가**(등록·생성·만들기) 하려면 다음 절차를 따릅니다. 세부 일정은 프로젝트 일정 아래의 더 작은 할 일입니다.

1. 간트차트에서 특정 프로젝트 일정 막대 옆의 **`+일정`** 버튼(또는 행 끝 작은 ➕ 아이콘) 을 누릅니다.
2. **"세부일정 생성"** 모달이 열립니다. 입력 항목은 프로젝트 일정과 유사 (제목·시작·종료·진행률·담당자·색상).
3. **`생성`** → 상위 프로젝트 일정 아래 들여쓰기 형태로 표시됩니다.

세부 일정은 간트차트에서 들여쓰기로 표시되며, 상위 프로젝트 일정과 색상·진행률을 독립적으로 관리합니다.

## 프로젝트 수정(편집·변경) — 단계별 (만든 사람만)

프로젝트를 **수정**(편집·변경) 하려면 본인이 만든 프로젝트를 선택해 다음 절차를 따릅니다.

1. 수정할 프로젝트를 선택한 상태에서 좌측 상단의 **`프로젝트 수정`** (연필 아이콘) 을 누릅니다.
2. **"프로젝트 수정"** 모달에 현재 값이 채워져 있습니다.
3. 변경 후 **`저장`**.

프로젝트 일정 / 세부 일정 수정도 동일 패턴 — 막대를 클릭하면 상세 모달이 열리고, 만든 사람이면 **`수정`** 버튼이 표시됩니다.

### 수정 권한
- 프로젝트·프로젝트 일정·세부 일정 모두 **만든 사람 본인만** 수정 가능.
- 팀장이라도 다른 팀원이 만든 것을 임의로 고칠 수 없습니다 (생성자 기반 권한).

## 프로젝트 삭제(제거·지우기) — 단계별 (만든 사람만)

프로젝트를 **삭제**(제거·지우기) 하려면 본인이 만든 프로젝트를 선택해 다음 절차를 따릅니다.

1. 삭제할 프로젝트를 선택한 상태에서 **`프로젝트 삭제`** (휴지통 아이콘) 을 누릅니다.
2. 확인 대화상자 → `확인` → 영구 삭제.

### CASCADE 삭제
프로젝트 삭제 시 그 프로젝트의 **모든 하위 데이터** 가 연쇄 삭제됩니다:
- 프로젝트 일정 전부
- 그 아래의 세부 일정 전부
- 프로젝트 전용 채팅 메시지·공지
- 프로젝트 전용 자료실 글·첨부파일

복구 불가. 신중히 결정하세요. 다른 팀원이 작업 중인 프로젝트라면 사전에 채팅으로 알리는 것을 권장합니다.

### 프로젝트 일정·세부 일정 단독 삭제
간트차트에서 해당 막대 클릭 → 상세 모달 → 본인이 만든 것이면 **`삭제`** 버튼. 세부 일정 삭제는 상위 프로젝트 일정에 영향 없음. 프로젝트 일정 삭제 시 그 아래 세부 일정도 함께 CASCADE 삭제.

## 진행률 (progress) 관리

각 일정의 진행률은 **0~100% 사이의 정수값으로 직접 입력** 합니다. 시스템이 자동 계산하지 않으며, 담당자가 진척에 따라 수동으로 갱신합니다.

- 프로젝트 자체 진행률 — 별도 입력
- 프로젝트 일정 진행률 — 별도 입력 (상위 프로젝트와 무관)
- 세부 일정 진행률 — 별도 입력

간트차트의 막대 안이 진행률만큼 채워져 시각화됩니다.

## 지연 표시 (isDelayed)

지연 여부는 사용자가 직접 표시합니다. "오늘 날짜가 종료 날짜를 지났고 진행률이 100% 가 아니면 자동 지연" 같은 자동 계산은 현재 미지원입니다. 담당자가 상황을 보고 수동으로 isDelayed 를 켜야 강조 표시됩니다.

## 간트차트 SVG 저장(다운로드·내보내기)

간트차트 화면 상단의 **`저장`** 버튼을 누르면 현재 간트차트 화면이 SVG 파일로 다운로드 폴더에 저장됩니다.

- 파일명: **`{프로젝트명}_{연도}.svg`** (예: `2분기마케팅캠페인_2026.svg`)
- 다크 모드 배경색까지 그대로 적용되어 보고서·문서 첨부에 바로 쓸 수 있습니다.
- 브라우저에서 즉시 처리되며 **서버·DB에는 아무것도 저장하지 않습니다.** 권한 제한 없이 화면을 볼 수 있는 팀 구성원이면 누구나 저장할 수 있습니다.

> 모바일 화면에서는 저장 버튼이 표시되지 않습니다. PC(넓은 화면)에서 사용하세요.

## 자주 묻는 질문

### Q. 프로젝트는 누가 만들 수 있나요?
팀 구성원이라면 누구나 (LEADER·MEMBER 모두) 만들 수 있습니다. 프로젝트 일정·세부 일정도 마찬가지.

### Q. 프로젝트 종료 날짜가 시작 날짜보다 앞이면 어떻게 되나요?
저장 시 오류가 납니다. 프로젝트·프로젝트 일정·세부 일정 모두 **종료 날짜 ≥ 시작 날짜** 여야 합니다 (`end_date >= start_date`). 일정과 다르게 같은 날짜는 허용됩니다 (당일 시작·당일 종료 가능).

### Q. 프로젝트 채팅방은 어떻게 사용하나요?
프로젝트 페이지로 들어가면 우측 채팅이 자동으로 그 프로젝트 전용방으로 전환됩니다. 일자별 팀 채팅과 별도로 관리되어, 그 프로젝트만의 메시지·공지·자료실 글이 따로 정리됩니다. 프로젝트 페이지를 벗어나면 우측 채팅도 일자별 팀 채팅으로 돌아옵니다.

### Q. 다른 팀의 프로젝트도 보이나요?
아니요. 한 팀의 프로젝트·간트차트·프로젝트 채팅·자료실은 그 팀 안에서만 보입니다.

### Q. AI 찰떡이로 프로젝트 등록·수정 가능한가요?
아니요. 찰떡이는 **일정** 조회·등록·수정·삭제만 처리합니다. 프로젝트는 구조가 복잡해서(프로젝트 → 프로젝트 일정 → 세부 일정 + 단계·담당자·진행률) 자연어 자동화 위험이 큽니다. 프로젝트 화면에서 직접 처리하세요.

### Q. 색상은 어떻게 활용하나요?
프로젝트 일정·세부 일정의 색상은 5가지 중 사용자가 선택합니다. 간트차트에서 시각적 구분에 사용됩니다. 의미는 정해져 있지 않으니 팀 자체 규약 ("긴급은 rose, 일반은 amber") 으로 활용하세요.
