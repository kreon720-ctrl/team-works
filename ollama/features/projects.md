# 프로젝트 · 간트차트 · 세부 일정

## 프로젝트는 무엇인가요

프로젝트는 업무의 **큰 틀**입니다. "2분기 신제품 런칭", "사이트 리뉴얼", "여름 워크샵 준비" 같은 단위로 만듭니다. 프로젝트 안에는 더 작은 **프로젝트 일정**(간트차트의 한 줄)을 두고, 그 아래 다시 **세부 일정**을 넣는 3단계 구조입니다.

```
프로젝트 (Project)
 └── 프로젝트 일정 (ProjectSchedule) ← 간트차트의 한 줄
      └── 세부 일정 (SubSchedule)    ← 들여쓰기로 표시
```

프로젝트는 다음 정보를 가집니다:
- **이름** (필수, 최대 200자)
- **설명** (선택)
- **시작 날짜 / 종료 날짜** (종료 ≥ 시작)
- **진행률** (0~100%)
- **담당자 (manager)**
- **단계 (phases)** — "기획 → 개발 → 출시" 같은 사용자 정의 배열

각 프로젝트는 **독립된 채팅방·공지·자료실**을 가집니다. 프로젝트를 열면 우측 채팅 영역이 그 프로젝트 전용 컨텍스트로 자동 전환되어, 일자별 팀 채팅과는 따로 관리됩니다.

## 프로젝트는 어떻게 만드나요

1. 팀 메인 화면에서 **프로젝트** 탭으로 들어갑니다 (또는 좌측 캘린더 뷰에서 프로젝트 모드 선택).
2. **+ 프로젝트 만들기** 버튼을 누릅니다.
3. 이름·설명·시작·종료 날짜·담당자·단계를 입력합니다.
4. **저장** → 프로젝트 목록에 추가됩니다.

## 프로젝트의 "단계(phases)"는 무엇인가요

프로젝트가 어떤 흐름으로 진행되는지 사용자가 정의하는 자유 배열입니다. 예시:
- 신제품 런칭: "기획 → 디자인 → 개발 → QA → 출시"
- 워크샵: "장소 섭외 → 프로그램 기획 → 참가자 모집 → 운영"

단계는 시각적 구분용이며, 실제 일정 진행은 그 아래 **프로젝트 일정**으로 관리합니다.

## 간트차트는 어디에서 보나요

프로젝트 목록에서 특정 프로젝트를 선택하면 **간트차트 뷰**가 열립니다. 가로축은 날짜, 세로축은 프로젝트 일정 목록입니다. 각 일정이 막대 형태로 표시되고, 진행률만큼 막대가 채워지며, 지연된 일정은 강조 표시됩니다.

간트차트 뷰가 활성화되면 우측 채팅 영역도 그 프로젝트 전용 채팅방으로 자동 전환됩니다 — 그 프로젝트만의 메시지·공지·자료실이 보입니다.

## 프로젝트 일정(ProjectSchedule)은 무엇인가요

프로젝트 안의 구체적인 업무 덩어리. 간트차트에서 한 줄(row)로 표시됩니다. 각 일정은 다음 정보를 가집니다:
- **제목** (필수, 최대 200자)
- **시작 날짜 / 종료 날짜** (종료 ≥ 시작)
- **진행률** (0~100%)
- **지연 여부** (isDelayed — 지연된 경우 강조 표시)
- **담당자 (leader)**
- **색상** (5가지 — indigo, blue, emerald, amber, rose)

## 세부 일정(SubSchedule)은 무엇인가요

프로젝트 일정을 더 작게 쪼갠 할 일입니다. 예: "콘텐츠 제작" 프로젝트 일정 아래에 "SNS 콘텐츠 초안", "썸네일 디자인", "최종 검수" 같은 세부 일정을 둘 수 있습니다.

세부 일정도 시작·종료 날짜, 진행률, 담당자, 색상을 가집니다. 간트차트에서 상위 프로젝트 일정 아래 들여쓰기 형태로 표시됩니다.

## 프로젝트는 누가 만들 수 있나요

팀 구성원이라면 누구나(LEADER·MEMBER 모두) 만들 수 있습니다. 프로젝트 일정·세부 일정 역시 팀원 누구나 만들 수 있습니다.

## 프로젝트 수정/삭제는 누가 할 수 있나요

**만든 사람 본인만** 수정·삭제할 수 있습니다. 팀장이라도 다른 팀원이 만든 프로젝트를 임의로 고치거나 지울 수 없습니다 (생성자 기반 권한).

## 프로젝트를 삭제하면 하위 일정도 같이 삭제되나요

네. 프로젝트를 삭제하면 그 안의 **모든 프로젝트 일정과 세부 일정이 연쇄 삭제(CASCADE)** 됩니다. 마찬가지로 그 프로젝트의 채팅 메시지·공지·자료실 글·첨부파일도 함께 정리됩니다. 삭제는 신중하게 결정하세요 — 복구할 수 없습니다.

## 진행률(progress)은 어떻게 관리하나요

각 일정의 진행률은 0~100% 사이의 정수값으로 직접 입력합니다. 시스템이 자동 계산하지 않으며, 담당자가 진척에 따라 수동으로 갱신합니다. 간트차트의 막대 안이 진행률만큼 채워져 시각화됩니다.

## 지연 표시(isDelayed)는 자동으로 되나요

지연 여부는 사용자가 직접 표시합니다. "오늘 날짜가 종료 날짜를 지났고 진행률이 100%가 아니면 자동 지연" 같은 자동 계산은 현재 미지원입니다. 담당자가 상황을 보고 수동으로 isDelayed를 켜야 강조 표시됩니다.

## 색상은 어떻게 활용하나요

프로젝트 일정·세부 일정의 색상은 5가지 중 사용자가 선택합니다. 간트차트에서 시각적 구분에 사용됩니다. 의미는 정해져 있지 않으니 팀 자체 규약("긴급은 rose, 일반은 amber" 등)으로 활용하세요.

## 다른 팀의 프로젝트도 보이나요

아니요. 한 팀의 프로젝트·간트차트·프로젝트 채팅·자료실은 그 팀 안에서만 보입니다. 팀 간 격리는 일정·채팅과 동일합니다.

## 프로젝트 종료 날짜가 시작 날짜보다 앞이면 어떻게 되나요

저장 시 오류가 납니다. 프로젝트·프로젝트 일정·세부 일정 모두 **종료 날짜가 시작 날짜보다 같거나 뒤**여야 합니다 (`end_date >= start_date`). 일정과 다르게 같은 날짜는 허용됩니다 (당일 시작·당일 종료 가능).
