# 포스트잇 (날짜 메모)

## 포스트잇이란 무엇인가요

캘린더의 특정 날짜에 붙이는 짧은 메모입니다. 일정과는 별개로 관리됩니다. 시작·종료 시각이 없고 날짜 단위로만 관리됩니다. "오늘 우유 사오기", "이번 주 금요일까지 보고서 마감" 같은 가벼운 알림에 적합합니다.

포스트잇에는 다음 정보가 있습니다:
- **날짜** (필수, YYYY-MM-DD 형식)
- **색상** (5가지 중 — indigo, blue, emerald, amber, rose)
- **내용** (텍스트)

## 포스트잇 vs 일정 vs 자료실

| 항목 | 일정 (Schedule) | 포스트잇 (Postit) | 자료실 (Board) |
|------|-----------------|-------------------|---------------|
| 시간 | 시작·종료 시각 필수 | 날짜만 | 작성 시각만 |
| 길이 | 분/시간 단위 표현 | 단순 메모 | 글 + 첨부 |
| 용도 | 회의·마감·이벤트 | 가벼운 알림·할일 | 회의록·자료·보고서 |
| 표시 | 캘린더 셀에 막대 | 캘린더 셀에 색상 메모 | 채팅 sub-탭의 글 목록 |
| 첨부 | 없음 | 없음 | 1개, ≤10MB |
| 글자 수 | 제목 200자 + 설명 | 텍스트 1개 | 제목 200자 + 본문 20000자 |

회의나 마감처럼 "몇 시부터 몇 시까지"가 중요한 것은 일정으로, "그 날 챙겨야 할 잡일"은 포스트잇으로, "팀 전체가 두고두고 보는 자료/회의록"은 자료실 글로 쓰는 게 일반적입니다.

## 포스트잇은 어떻게 만드나요

1. 캘린더에서 원하는 날짜를 선택합니다.
2. **+ 포스트잇 추가** 버튼을 누릅니다 (위치는 화면 디자인에 따라 다름).
3. 색상을 5가지 중 하나 선택하고 내용을 입력합니다.
4. **저장** → 그 날짜 캘린더 셀에 즉시 표시됩니다.

## 포스트잇은 누가 만들 수 있나요

팀 구성원이라면 누구나(LEADER·MEMBER 모두) 만들 수 있습니다. 일정과 같은 정책입니다.

## 포스트잇 수정/삭제는 누가 할 수 있나요

**만든 사람 본인만** 수정·삭제할 수 있습니다. 팀장이라도 다른 팀원의 포스트잇을 임의로 고치지 못합니다 (생성자 기반 권한). 본인이 만든 포스트잇에는 [수정]/[삭제] 옵션이 표시되고, 다른 사람 것에는 표시되지 않습니다.

## 포스트잇 색상은 몇 가지인가요

5가지 — **indigo**(남색), **blue**(파랑), **emerald**(에메랄드), **amber**(앰버/주황), **rose**(장미/분홍). 일정·프로젝트 일정과 같은 색상 팔레트입니다.

색상의 의미는 시스템이 정하지 않습니다. 팀 자체 규약("긴급은 rose, 일반은 amber")으로 활용하세요.

## 한 날짜에 포스트잇을 여러 개 붙일 수 있나요

네. 같은 날짜에 여러 개의 포스트잇을 붙일 수 있습니다. 같은 사용자가 여러 개를 만들어도 되고, 여러 팀원이 각자 만들어도 됩니다. 캘린더 셀에 작은 색상 메모가 차곡차곡 표시됩니다.

## 포스트잇이 다른 날짜로 이동하나요

자동 이동은 없습니다. 한 번 붙인 날짜에 그대로 남습니다. 날짜를 바꾸려면 수정 기능으로 다른 날짜를 지정하세요 (만든 사람 본인만 가능).

## 포스트잇은 다른 팀에도 보이나요

아니요. 그 팀 안에서만 보입니다. 팀 단위 격리는 일정·채팅·자료실과 동일하게 적용됩니다.

## 포스트잇 알림이 오나요

현재 MVP 버전에는 포스트잇 푸시 알림 기능이 없습니다. 캘린더에 들어와서 시각적으로 확인하는 방식입니다.
