# 자료실 (게시판)

## 자료실은 무엇인가요

채팅방 sub-탭으로 붙어 있는 글 + 첨부파일 공유 공간입니다. 채팅에 파일을 올리면 며칠만 지나도 스크롤 속에 묻혀 찾기 어려운 문제를 해결합니다. 회의록, 보고서, 자료, 이미지 등을 **제목 + 본문 + 첨부파일** 형태로 정리해 두면 누구나 깔끔하게 찾을 수 있습니다.

## 자료실은 어디에서 보나요

팀 메인 화면(`/teams/[팀ID]`) 우측 채팅 영역 상단의 **팀채팅** 탭 안에 다시 sub-탭으로 **채팅 / 자료실** 토글이 있습니다. **자료실** sub-탭을 누르면 글 목록이 표시됩니다.

## 일자별 자료실과 프로젝트 자료실의 차이

채팅·공지와 동일하게 자료실도 **(팀, 프로젝트) 격리** 됩니다.

- **팀 일자별 자료실** — 좌측 캘린더가 월·주·일 뷰일 때 우측에 표시되는 팀 공통 자료실. 팀 전체가 함께 보는 자료.
- **프로젝트 전용 자료실** — 좌측이 프로젝트 간트차트 뷰일 때 우측에 자동 전환되는, 그 프로젝트만의 자료실.

두 컨텍스트는 서로 섞이지 않습니다. 프로젝트 A 의 자료실 글이 팀 일자별 자료실이나 프로젝트 B 자료실에 안 보입니다.

## 글은 어떻게 작성하나요

1. 자료실 sub-탭에서 **+ 글쓰기** 버튼을 누릅니다.
2. 다음을 입력합니다:
   - **제목** (필수, 1~200자)
   - **본문** (선택, 최대 20000자, 줄바꿈 보존)
   - **첨부파일** (선택, 1개, ≤10MB)
3. **등록** → 자료실 목록에 즉시 추가됩니다.

## 글 작성은 누가 할 수 있나요

팀 구성원이라면 누구나(LEADER·MEMBER 모두) 작성 가능합니다.

## 글 수정·삭제는 누가 할 수 있나요

**작성자 본인만** 수정·삭제할 수 있습니다. 팀장이라도 다른 팀원의 자료실 글을 임의로 고치거나 지울 수 없습니다 (생성자 기반 권한 정책). 본인이 쓴 글의 상세 화면에는 우상단에 [수정][삭제] 버튼이 보이고, 다른 사람 글에는 표시되지 않습니다.

## 글을 삭제하면 첨부파일도 함께 삭제되나요

네. 글을 삭제하면 그 글에 붙은 첨부파일도 디스크(또는 클라우드 스토리지)에서 함께 정리됩니다(CASCADE). 한 번 삭제하면 복구할 수 없습니다.

## 첨부파일 크기 제한은 얼마인가요

**1개 파일 ≤ 10MB**. 1단계 버전은 글당 첨부 1개까지만 가능합니다. 향후 다중 첨부로 확장 예정이며, DB 모델은 이미 1:N 구조라 UI 만 확장하면 됩니다.

11MB 이상의 파일을 업로드하면 거부됩니다 (HTTP 413).

## 첨부파일 형식은 어떤 게 허용되나요

다음 MIME 화이트리스트만 통과합니다:
- 이미지 — `jpg`, `png`, `gif`, `webp`
- 문서 — `pdf`, `docx`, `xlsx`, `pptx`, `txt`, `md`
- 압축 — `zip`

다음은 **거부** 됩니다:
- **SVG** — XSS 위험으로 차단 (이미지 카테고리에서 명시적 제외)
- **실행파일** — `.exe`, `.dll`, `.bat`, `.sh` 등
- **HTML** — 인라인 렌더 위험으로 차단
- 화이트리스트에 없는 모든 형식

확장자뿐 아니라 파일의 첫 8바이트(매직 바이트)를 다시 검증하므로, 확장자만 `.png`로 바꾼 실행파일 같은 우회 시도는 차단됩니다.

## 다른 사람이 올린 자료실 첨부파일은 어떻게 받나요

자료실 글 상세 화면에서 첨부파일 항목을 클릭하면 다운로드됩니다. 응답 헤더에 `Content-Disposition: attachment` 가 강제로 붙어 브라우저가 인라인 렌더하지 않고 항상 파일로 저장됩니다.

## 다른 팀의 자료실 첨부파일도 받을 수 있나요

아니요. 첨부파일 다운로드 시 백엔드가 `첨부 → 글 → 팀 → 사용자 멤버십` 을 검증하므로, 다른 팀의 첨부파일은 다운로드되지 않습니다 (HTTP 403).

## 자료실 글에 댓글을 달 수 있나요

현재 MVP 버전에는 댓글 기능이 없습니다. 글 본문에 추가 정보가 필요하면 작성자가 직접 글을 수정해야 합니다. 향후 추가 예정 기능입니다.

## 자료실에 검색 기능이 있나요

현재 MVP 버전에는 검색·페이지네이션 기능이 없습니다. 글 목록은 작성 시각 역순(최신 글 위)으로 표시됩니다. 향후 추가 예정 기능입니다.

## 글 본문에 마크다운이나 리치 텍스트를 쓸 수 있나요

현재 MVP 버전은 **plain text + 줄바꿈 보존** 만 지원합니다. 본문은 입력한 그대로 표시되며, `**굵게**` 같은 마크다운 문법은 그대로 글자로 보입니다. 향후 마크다운·리치 텍스트로 확장될 수 있습니다.

## 자료실 자료는 어디에 저장되나요

1단계 운영에서는 **호스트 디스크의 `files/` 디렉토리** 에 저장되며, Docker 컨테이너 재기동·재배포 후에도 유지됩니다(volume mount). DB에는 메타데이터(원본 파일명, 식별자, MIME, 크기)만 저장됩니다.

운영 환경에서는 환경 변수 한 줄(`STORAGE_BACKEND=s3`)만 토글하면 클라우드 스토리지(S3 등)로 무중단 이전이 가능하며, 호출처 코드는 한 줄도 바뀌지 않습니다. 사용자 입장에서 저장 위치 변화는 보이지 않습니다.

## 첨부파일은 영구 보관되나요

네. 글이 삭제되기 전까지는 영구 저장됩니다. 글 수정 시 새 파일을 첨부하면 기존 파일은 자동 정리되고 새 파일로 교체됩니다 (1단계는 글당 첨부 1개라 교체 방식).

## 자료실 목록은 얼마나 자주 갱신되나요

채팅(3초 폴링)보다 느슨한 **약 10초 주기** 로 자동 새로고침됩니다. 글 자체는 채팅처럼 자주 변하지 않으므로 폴링 부담을 줄였습니다.
