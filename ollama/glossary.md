# 용어 사전

## 팀 (Team)

사용자들이 협업하는 기본 단위. 이름(최대 100자), 설명(선택, 최대 500자), 공개 여부를 가집니다. 팀을 만든 사람은 자동으로 그 팀의 **팀장(LEADER)**이 됩니다. 공개로 설정하면 로그인한 모든 사용자가 팀 탐색 화면에서 보고 가입 신청할 수 있습니다. 한 팀의 일정·채팅·포스트잇·프로젝트·자료실은 그 팀 안에서만 보이며, 다른 팀에는 절대 노출되지 않습니다.

## 팀장 (LEADER)

팀을 만든 사용자 또는 팀의 최고 권한자. 팀 1개당 1명입니다. 팀장이 할 수 있는 일:
- 팀 가입 신청 승인 또는 거절
- 업무보고(WORK_PERFORMANCE) 메시지 열람 — 항상 가능
- 다른 팀원에게 업무보고 열람 권한 부여
- 다른 팀원이 작성한 공지사항도 삭제 가능 (팀장 예외 권한)
- 팀 정보 수정 (이름·설명·공개 여부)
- 팀 삭제 (하위 데이터 모두 CASCADE 삭제)
- 팀원 강제 탈퇴 (본인 제외)

팀장이 **할 수 없는 일**: 다른 팀원이 만든 일정·포스트잇·프로젝트·일반 메시지·자료실 글을 수정·삭제하는 것. 본인이 만든 것만 가능합니다.

## 팀원 (MEMBER)

팀에 가입된 일반 사용자. 팀장이 가입 신청을 승인하면 MEMBER가 됩니다. 팀원이 할 수 있는 일:
- 팀 일정·포스트잇·프로젝트·공지사항·자료실 글 작성
- 일반 채팅 / 업무보고 / 프로젝트 전용 채팅 송수신
- 본인이 만든 콘텐츠 수정·삭제

팀원이 **할 수 없는 일**: 팀장 승인 권한, 업무보고 열람 권한 설정, 팀 정보 수정·삭제, 다른 사람이 만든 콘텐츠 수정·삭제.

## 가입 신청 (TeamJoinRequest)

비구성원이 공개 팀에 들어가기 위해 제출하는 신청. 상태는 세 가지:
- **PENDING (대기)** — 신청 직후 상태. 팀장의 응답을 기다림
- **APPROVED (승인)** — 팀장이 승인. 자동으로 MEMBER 등록
- **REJECTED (거절)** — 팀장이 거절. 팀 합류 안 됨. 다시 신청 가능

같은 팀에 PENDING 신청을 동시에 두 개 이상 가질 수 없습니다. 이미 멤버이면 신청 자체가 막힙니다.

## 일정 (Schedule)

팀 캘린더에 표시되는 이벤트. 제목(필수, 최대 200자), 설명(선택), 시작 시각, 종료 시각, 색상(5가지). **종료 시각은 반드시 시작 시각보다 뒤**여야 합니다. 팀원 누구나 만들 수 있지만 수정·삭제는 만든 사람 본인만 가능합니다.

## 포스트잇 (Postit)

특정 날짜에 붙이는 짧은 메모. 일정과 별개로 시간 정보 없이 날짜 단위로 관리됩니다. 색상 5가지 중 선택. 팀원 누구나 만들 수 있고, 수정·삭제는 본인만 가능합니다.

## 채팅 메시지 (ChatMessage)

팀 채팅에서 주고받는 메시지. 두 가지 타입과 두 가지 컨텍스트가 있습니다:

타입:
- **NORMAL (일반)** — 보통 대화. 팀원 모두 조회 가능
- **WORK_PERFORMANCE (업무보고)** — 팀원이 팀장에게 보내는 보고. 기본은 팀장만 조회

컨텍스트(`projectId` 컬럼):
- **팀 일자별** — `projectId NULL`. KST 날짜로 그룹핑
- **프로젝트 전용** — `projectId 채워짐`. 해당 프로젝트 채팅방에서만 노출

내용은 최대 2000자.

## 업무보고 (WORK_PERFORMANCE)

특별한 채팅 메시지 타입. 팀원이 팀장에게 보내는 평가성·민감한 보고를 위한 분리된 채널입니다.
- 작성한 본인은 항상 자기 메시지를 봅니다
- 팀장은 모든 업무보고를 봅니다
- 다른 팀원은 **기본적으로 보지 못합니다**
- 팀장이 권한을 부여한 팀원만 다른 사람의 업무보고를 볼 수 있습니다

## 업무보고 열람 권한 (WorkPerformancePermission)

팀장이 특정 팀원에게 "다른 사람의 업무보고를 봐도 좋다"라고 부여하는 권한. 팀장만 설정·변경 가능. 빈 상태(아무도 권한 없음)면 팀장만 업무보고를 봅니다.

## 공지사항 (Notice)

채팅 상단에 고정되어 표시되는 알림. 팀 일자별 / 프로젝트 전용 두 컨텍스트 모두 가질 수 있습니다(채팅과 동일한 격리 패턴). 최대 2000자. 팀원 누구나 작성 가능, 삭제는 작성자 본인 또는 팀장.

## 프로젝트 (Project)

업무의 큰 틀. 시작·종료 날짜(종료 ≥ 시작), 진행률(0~100), 담당자, 단계(phases — 사용자 정의 배열)를 가집니다. 팀원 누구나 만들 수 있고 수정·삭제는 본인만 가능. 프로젝트마다 **독립된 채팅·공지·자료실** 을 가집니다.

## 프로젝트 일정 (ProjectSchedule)

프로젝트 안의 구체적인 업무 덩어리. 간트차트의 한 줄로 표시됩니다. 시작·종료 날짜, 진행률, 지연 여부(isDelayed), 담당자(leader), 색상(5가지)을 가집니다.

## 세부 일정 (SubSchedule)

프로젝트 일정을 더 잘게 쪼갠 할 일. 간트차트에서 상위 일정 아래 들여쓰기로 표시됩니다.

## 자료실 글 (BoardPost)

채팅방 sub-탭으로 노출되는 글. 제목(1~200자), 본문(≤20000자), 첨부파일(≤10MB)을 가집니다. 팀 일자별 / 프로젝트 전용 두 컨텍스트 모두 가질 수 있습니다(채팅·공지와 동일 격리). 작성은 팀원 누구나, 수정·삭제는 작성자 본인만 가능합니다.

## 자료실 첨부파일 (BoardAttachment)

자료실 글에 첨부된 파일. 1단계는 글당 1개까지. 크기 ≤ 10MB. MIME 화이트리스트(jpg, png, gif, webp, pdf, docx, xlsx, pptx, txt, md, zip)에 통과해야 하며 매직 바이트로 한 번 더 검증합니다. SVG·실행파일·HTML 등은 거부됩니다. 다운로드는 같은 팀 멤버만 가능하며 응답 헤더에 `Content-Disposition: attachment` + `X-Content-Type-Options: nosniff` 가 강제 부여되어 인라인 렌더가 차단됩니다.

## 캘린더 뷰 (월/주/일/프로젝트)

팀 캘린더는 네 가지 뷰로 볼 수 있습니다:
- **월 뷰** — 한 달 전체. 일정과 포스트잇이 날짜 셀에 표시
- **주 뷰** — 7일치 시간대별
- **일 뷰** — 하루 시간대별 상세
- **프로젝트 뷰** — 프로젝트 간트차트. 활성 시 우측 채팅 영역이 해당 프로젝트 채팅방으로 자동 전환

## JWT 토큰

로그인 시 발급되는 인증 정보:
- **Access Token (15분)** — 모든 요청에 포함되는 짧은 수명 토큰
- **Refresh Token (7일)** — Access Token이 만료되면 갱신용으로 사용

7일 동안 한 번도 활동 없으면 Refresh Token이 만료되어 다시 로그인이 필요합니다.

## AI 버틀러 "찰떡"

TEAM WORKS 사용자를 돕는 AI 도우미. 우측 채팅 영역의 **AI 버틀러** 탭에서 단일 입력창으로 호출합니다. 자연어 질문을 받아 자동으로 6가지 길 중 하나로 분류해 답합니다:

| 분류 | 처리 경로 | 답변 형태 |
|------|---------|---------|
| **usage (사용법)** | RAG (`ollama/*.md` 인덱스) | 공식 문서 인용 + 📚 출처 뱃지 |
| **general (일반)** | SearxNG 메타검색 + Open WebUI gemma4:26b | 웹 검색 5건 인용 + 🌐 출처 뱃지 |
| **schedule_query (일정 조회)** | backend Schedule API 직접 조회 | 코드가 한국어로 즉시 포맷 |
| **schedule_create (일정 등록)** | confirm 카드 + 사용자 ✓ 후 INSERT | 다중 턴 — 정보 부족 시 후속 질문 |
| **schedule_delete (일정 삭제)** | confirm 카드 + 사용자 ✓ 후 DELETE | 다중 후보 시 좁히기 안내, 벌크 차단 |
| **schedule_update (일정 수정)** | 식별 → 새 일시 → 새 제목 multi-step → confirm 카드 + ✓ 후 PATCH | 일시·제목만 지원, "그대로" 답변으로 부분 수정 |
| **blocked (거절)** | 정중한 거절 안내 | 일정 취소, 다른 도메인 변경, 비지원 필드(색깔·설명) 등 |

이전 명칭(AI 비서)과 동일한 도구입니다. 캐릭터 이름은 "찰떡" — "개떡같이 물어도 찰떡같이 대답한다"는 의미.

## 시간대 (Timezone)

서버 기본 시간대는 **KST (UTC+9)** 입니다. 일정·메시지·자료실 시각은 모두 KST 기준으로 보입니다. DB에는 UTC로 저장되지만 사용자는 의식할 필요 없습니다.
